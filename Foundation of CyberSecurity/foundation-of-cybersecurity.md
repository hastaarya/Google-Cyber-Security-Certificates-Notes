## Module 1: Welcome to Cybersecurity

### üîê Core Cybersecurity Concepts

**Cybersecurity Definition:**
Cybersecurity (also known as security) is the practice of ensuring confidentiality, integrity, and availability of information by protecting networks, devices, people, and data from unauthorized access or criminal exploitation.

**Key Terminology:**
- **Security Posture**: An organization's ability to manage its defense of critical assets and data and react to change
- **Threat Actor**: Any person or group who presents a security risk
- **Internal Threat**: A current or former employee, external vendor, or trusted partner who poses a security risk
- **Compliance**: The process of adhering to internal standards and external regulations

**Specialized Security Areas:**
- **Network Security**: Practice of keeping an organization's network infrastructure secure from unauthorized access
- **Cloud Security**: Process of ensuring cloud assets are properly configured and access is limited to authorized users
- **Application Security**: Protecting applications from threats throughout their lifecycle

### üéØ Types of Cyber Attacks

**Social Engineering Attacks:**
- **Phishing**: Deceptive emails pretending to be from legitimate sources
- **Spear Phishing**: Targeted phishing against specific individuals or organizations
- **Whaling**: Phishing attacks targeting senior executives
- **Vishing**: Voice phishing using phone calls
- **Smishing**: SMS/text message phishing
- **Business Email Compromise (BEC)**: Impersonating executives for financial fraud
- **Social Media Phishing**: Using social platforms to gather information and launch attacks

**Technical Attack Methods:**
- **Password Attacks**:
  - Brute Force: Trying all possible password combinations
  - Dictionary Attacks: Using word lists
  - Rainbow Table Attacks: Using precomputed hash tables
- **Malware Types**:
  - Viruses: Self-replicating malicious code
  - Worms: Network-spreading malware
  - Trojans: Disguised as legitimate software
  - Ransomware: Encrypts files for ransom
  - Spyware: Secretly monitors user activity
- **Network Attacks**:
  - DDoS Attacks: Overwhelming services with traffic
  - Man-in-the-Middle: Intercepting communications
  - DNS Spoofing: Redirecting domain requests

**Advanced Attack Vectors:**
- **Supply Chain Attacks**: Compromising software/hardware through vendors
- **Cryptographic Attacks**: Exploiting encryption weaknesses
- **Physical Attacks**: Malicious USB devices, card skimming
- **Adversarial AI**: Manipulating machine learning systems
- **Zero-Day Exploits**: Attacking unknown vulnerabilities

### üë§ Threat Actor Profiles

**Advanced Persistent Threats (APTs):**
- **Characteristics**: Highly sophisticated, well-funded, patient attackers
- **Motivations**: Espionage, intellectual property theft, infrastructure damage
- **Targets**: Government agencies, large corporations, critical infrastructure
- **Examples**: Nation-state actors, organized cybercrime groups

**Insider Threats:**
- **Types**:
  - Malicious Insiders: Disgruntled employees seeking revenge
  - Negligent Insiders: Careless employees causing accidental breaches
  - Compromised Insiders: Credentials stolen by external attackers
- **Detection Challenges**: Legitimate access makes detection difficult
- **Prevention**: Access controls, monitoring, employee training

**Hacktivist Groups:**
- **Motivations**: Political agendas, social change, activism
- **Methods**: Website defacement, data leaks, DDoS attacks
- **Examples**: Anonymous, LulzSec

**Cybercriminal Organizations:**
- **Motivations**: Financial gain through fraud, extortion, theft
- **Methods**: Ransomware, phishing, business email compromise
- **Operations**: Often run as business-like organizations

## Module 3: Protect Against Threats

### üõ°Ô∏è The CIA Triad

**Confidentiality:**
- **Definition**: Ensuring that only authorized users can access specific assets or data
- **Implementation Methods**:
  - Encryption (data at rest and in transit)
  - Access controls and authentication
  - Data classification
  - Network segmentation
- **Threats**: Eavesdropping, unauthorized access, data breaches

**Integrity:**
- **Definition**: Ensuring data is correct, authentic, and reliable
- **Implementation Methods**:
  - Hash functions and digital signatures
  - Version control systems
  - Change management procedures
  - Checksums and validation
- **Threats**: Data tampering, unauthorized modifications, corruption

**Availability:**
- **Definition**: Ensuring data and systems are accessible to authorized users when needed
- **Implementation Methods**:
  - Redundancy and failover systems
  - Backup and recovery procedures
  - DDoS protection
  - System monitoring
- **Threats**: DDoS attacks, system failures, ransomware

### üìä Security Frameworks

**NIST Cybersecurity Framework (CSF):**
- **Purpose**: Voluntary framework for managing cybersecurity risk
- **Core Functions**:
  1. **Identify** - Develop organizational understanding
  2. **Protect** - Implement appropriate safeguards
  3. **Detect** - Identify cybersecurity events
  4. **Respond** - Take action regarding detected incidents
  5. **Recover** - Maintain plans for resilience

**NIST Risk Management Framework (RMF):**
- **Six-Step Process**:
  1. Categorize information systems
  2. Select security controls
  3. Implement security controls
  4. Assess security controls
  5. Authorize information systems
  6. Monitor security controls

**Other Important Frameworks:**
- **ISO/IEC 27000 Series**: International information security standards
- **CIS Critical Security Controls**: 18 actionable security controls
- **COBIT**: Framework for governance and management of enterprise IT

### üìù Compliance Regulations

**Data Protection Regulations:**
- **GDPR (General Data Protection Regulation)**:
  - Scope: EU residents' data protection
  - Requirements: Consent, breach notification within 72 hours, data portability
  - Penalties: Up to 4% of global annual revenue

- **CCPA (California Consumer Privacy Act)**:
  - Scope: California residents' data rights
  - Requirements: Right to know, delete, opt-out
  - Enforcement: Civil penalties and private right of action

**Industry-Specific Regulations:**
- **HIPAA (Health Insurance Portability and Accountability Act)**:
  - Protected Health Information (PHI) security and privacy
  - Three rules: Privacy, Security, Breach Notification
  - Covered entities: Healthcare providers, insurers, clearinghouses

- **PCI DSS (Payment Card Industry Data Security Standard)**:
  - Credit card data protection
  - 12 requirements for cardholder data security
  - Applies to all organizations handling payment cards

- **SOX (Sarbanes-Oxley Act)**:
  - Financial reporting and corporate governance
  - CEO/CFO certification of financial statements
  - Internal controls assessment

**Critical Infrastructure Protection:**
- **NERC CIP**: North American electric reliability corporation critical infrastructure protection
- **FERC**: Federal Energy Regulatory Commission standards
- **CISA**: Cybersecurity and Infrastructure Security Agency guidelines

### üõ°Ô∏è Security Controls

**Control Types by Function:**
- **Preventive Controls**: Stop security incidents before they occur
  - Examples: Firewalls, access controls, encryption
- **Detective Controls**: Identify security incidents as they occur
  - Examples: IDS, log monitoring, security cameras
- **Corrective Controls**: Remediate damage from security incidents
  - Examples: Backups, patches, incident response
- **Deterrent Controls**: Discourage security violations
  - Examples: Warning signs, policies, security awareness

**Control Implementation Categories:**
- **Technical Controls**: Technology-based protection measures
  - Examples: Encryption, antivirus, authentication systems
- **Administrative Controls**: Policy and procedure-based measures
  - Examples: Security policies, training, background checks
- **Physical Controls**: Physical security measures
  - Examples: Locks, cameras, access badges

### ‚öñÔ∏è Security Ethics & Legal Considerations

**Professional Ethics Principles:**
- Maintain confidentiality and privacy
- Act with integrity and honesty
- Avoid conflicts of interest
- Continue professional development
- Protect society and the infrastructure

**Legal Considerations:**
- **Computer Fraud and Abuse Act (CFAA)**: US federal computer crime law
- **Electronic Communications Privacy Act**: Governs electronic communications
- **Cybersecurity Information Sharing Act**: Encourages threat intelligence sharing

**Incident Response Ethics:**
- **No Counterattacks**: Generally illegal and unethical
- **Evidence Preservation**: Maintain chain of custody
- **Responsible Disclosure**: Follow proper vulnerability reporting channels
- **Privacy Protection**: Balance investigation needs with individual rights

**International Considerations:**
- **Tallinn Manual**: Study of international law applicable to cyber warfare
- **Budapest Convention**: International treaty on cybercrime
- **Cross-Border Data Transfer**: Compliance with international data protection laws

### üîÑ Risk Management Process

**Risk Assessment Steps:**
1. **Asset Identification**: Catalog systems, data, and resources
2. **Threat Identification**: Identify potential threat sources
3. **Vulnerability Assessment**: Find weaknesses in systems
4. **Impact Analysis**: Determine potential business impact
5. **Risk Determination**: Calculate risk levels
6. **Control Recommendations**: Propose risk mitigation measures

**Risk Treatment Options:**
- **Avoidance**: Eliminate the risk entirely
- **Mitigation**: Implement controls to reduce risk
- **Transfer**: Shift risk to third party (insurance)
- **Acceptance**: Acknowledge and monitor the risk

## Module 4: Cybersecurity Tools

### üõ†Ô∏è Security Monitoring Tools

**SIEM (Security Information and Event Management):**
- **Purpose**: Collect, analyze, and correlate security event data
- **Key Capabilities**:
  - Real-time security monitoring
  - Log management and analysis
  - Threat detection and alerting
  - Compliance reporting
  - Incident investigation
- **Popular Solutions**: Splunk, IBM QRadar, ArcSight, AlienVault
- **Deployment Options**: On-premise, cloud, hybrid

**IDS/IPS (Intrusion Detection/Prevention Systems):**
- **IDS Types**:
  - Network-based (NIDS): Monitors network traffic
  - Host-based (HIDS): Monitors individual systems
- **Detection Methods**:
  - Signature-based: Known attack patterns
  - Anomaly-based: Behavioral deviations
  - Policy-based: Rule violations
- **IPS Functionality**: Can actively block malicious activity

**Network Protocol Analyzers:**
- **Purpose**: Capture and analyze network traffic
- **Common Tools**: Wireshark, tcpdump, Network Miner
- **Use Cases**:
  - Network troubleshooting
  - Security incident investigation
  - Protocol analysis
  - Performance monitoring

### üîß Security Tools & Technologies

**Vulnerability Management:**
- **Vulnerability Scanners**: Nessus, OpenVAS, Qualys
- **Configuration Management**: Ansible, Puppet, Chef
- **Patch Management**: WSUS, SCCM, automated deployment tools

**Network Security Tools:**
- **Firewalls**: Network segmentation and traffic filtering
- **VPNs**: Secure remote access
- **Web Application Firewalls (WAF)**: Protect web applications
- **Network Access Control (NAC)**: Device authentication and policy enforcement

**Encryption Technologies:**
- **Symmetric Encryption**: AES, DES (same key for encryption/decryption)
- **Asymmetric Encryption**: RSA, ECC (public/private key pairs)
- **Hashing Algorithms**: SHA-256, MD5 (data integrity verification)
- **Digital Certificates**: X.509 certificates for authentication

### üìã Security Documentation & Playbooks

**Incident Response Playbooks:**
- **Preparation Phase**: Tools, teams, and training
- **Identification Phase**: Detection and analysis
- **Containment Phase**: Short-term and long-term containment
- **Eradication Phase**: Remove malicious components
- **Recovery Phase**: Restore systems and services
- **Lessons Learned**: Post-incident analysis

**Digital Forensics Procedures:**
- **Chain of Custody**: Documenting evidence handling
- **Order of Volatility**: Preserving evidence from most to least volatile:
  1. CPU registers, cache
  2. Routing table, ARP cache, process table
  3. Memory (RAM)
  4. Temporary file systems
  5. Disk storage
  6. Remote logging and monitoring data
  7. Physical configuration, network topology
  8. Archival media

**Evidence Preservation:**
- **Forensic Imaging**: Bit-by-bit copies of storage media
- **Write Blockers**: Hardware/software to prevent evidence modification
- **Hashing**: Verify evidence integrity (MD5, SHA-256)
- **Documentation**: Detailed notes and photographs

## Key Cybersecurity Concepts

### üîç Threat Intelligence

**Types of Threat Intelligence:**
- **Strategic**: High-level trends for executive decision-making
- **Tactical**: TTPs (Tactics, Techniques, Procedures) of threat actors
- **Operational**: Specific impending attacks and campaigns
- **Technical**: Indicators of Compromise (IOCs)

**Threat Intelligence Sources:**
- **Open Source**: Publicly available information
- **Commercial**: Paid threat intelligence feeds
- **Government**: Official alerts and advisories
- **Community**: Information sharing organizations

### üéØ Security Architecture Principles

**Defense in Depth:**
- Multiple layers of security controls
- No single point of failure
- Diverse security technologies
- Redundant protective measures

**Zero Trust Architecture:**
- "Never trust, always verify"
- Assume breach mentality
- Micro-segmentation
- Identity-centric security
- Continuous verification

**Least Privilege Principle:**
- Users receive minimum access necessary
- Role-based access control (RBAC)
- Regular access reviews
- Just-in-time privilege elevation

**Risk Quantification:**
- **Single Loss Expectancy (SLE)**: Cost of single incident
- **Annual Rate of Occurrence (ARO)**: Incidents per year
